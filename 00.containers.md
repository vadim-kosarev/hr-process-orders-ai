# Развернуть следующие контейнеры в Docker:
- файл - docker-compose.yml, .env в корне проекта
- версии - предпочтительнее официальные с тегом latest, если не указано иное

1. REDIS (single instance), REDIS-UI (redis-insight)
2. KAFKA (single instance) image: Apache Kafka, Без zookeeper, KRaft mode; KAFKA-UI: provectuslabs/kafka-ui:latest
3. PostgreSQL (single instance)
4. Spring-boot application java 21 - в контейнере развернут только результат локального билда - .jar файл - посмотри в build.gradle.kts какой именно. В docker-compose.yml - volume на .jar-файл в режиме dev.

- ДЛя каждого сервиса в контейнере настроить учетную запись для доступа по шаблону  service/service
- для БД и очередей предусмотреть файл с начальной структурой (например, для PostgreSQL - init.sql, для Kafka - начальная тема и т.д.) и подключить его к контейнеру при запуске.
