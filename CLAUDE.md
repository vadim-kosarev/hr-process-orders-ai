# Инструкции для ИИ-агента

Язык общения - русский
Язык комментариев в коде - английский

# Локальное рабочее окружение
- для run_command_tool: Windows PowerShell
  - команды "&&" нет, есть ";"
  - команды head нет, есть powershell аналоги
  - команды tail нет
  - команды grep нет
- используй запуск команд в терминале с выводом stderr 2>&1 для отладки и проверки ошибок
- в окружении есть python для утилитных скриптов, если нужно, но не для основной логики приложения


# Общие правила
- делать только то, что указано в инструкциях, и не выходить за их рамки.
- отчеты о работе - .md-файлы и прочая информация, сгенеренная AI-агентом должна находиться в папке .ai в корне проекта, нигде больше. к каждому файлу прибавлять префикс для отслеживания порядка их генерации.
- использовать историю изменений задокументированных в папке .ai
- файлы, относящиеся к docker-контейнерам должны располагаться в папке docker в корне проекта, кроме .env и docker-compose - они должны быть в корне проекта
- строго следовать правилу:
  - не генерировать тесты, если не указано явно

# Контейнеры
- docker-compose.yml - содержит конфигурацию для запуска всех необходимых сервисов, включая ИИ-агента, базу данных и другие компоненты.
- использовать .env файл для хранения конфиденциальных данных, таких как API-ключи и пароли.

# Java
- java version: 21
- соблюдать стандарты и лучшие практики Java, SOLID, DRY, KISS, Clean Code
- код - без комментариев. код самодкументируемый, понятный и читаемый. использовать говорящие названия переменных, методов, классов и т.д.
- для Spring компонентов (@Service, @Controller, @RestController, @Component, @Configuration и т.д.) обязательно использовать Lombok аннотации:
  - @RequiredArgsConstructor или @AllArgsConstructor для dependency injection через конструктор
  - @Slf4j для логирования
  - @Data, @Getter, @Setter для DTO и entity классов (где применимо)
  - избегать @Autowired полей, использовать constructor injection

# Тестирование
- для тестов используй те же контейнеры БД, брокеров и т.д., что и для основного приложения, чтобы обеспечить максимальную реалистичность тестов.
- тестовые данные по максимуму генери и читай из файлов для того, чтобы я мог их тоже использовать в ручных тестах, если нужно.
